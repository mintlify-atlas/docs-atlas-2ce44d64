ArchiveBox can import URLs from many different sources and formats. You can feed URLs one at a time, schedule regular imports, or bulk import from various services.

## Browser Extension

The [official browser extension](https://github.com/ArchiveBox/archivebox-extension) provides realtime archiving of browsing history or selected pages.

- **Supported browsers**: Chrome, Chromium, Brave, Edge, Firefox
- **Features**: Archive pages as you browse, or save selected pages on-demand
- **Download**: [Chrome Web Store](https://chromewebstore.google.com/detail/archivebox-exporter/habonpimjphpdnmcfkaockjnffodikoj)

## Manual URL Import

### Single URL

Archive a single URL directly from the command line:

```bash
archivebox add 'https://example.com/some/page'
```

### Multiple URLs from stdin

Pipe URLs from stdin:

```bash
echo 'http://example.com' | archivebox add

# Or multiple URLs
cat << EOF | archivebox add
https://news.ycombinator.com
https://example.com/article
EOF
```

**Docker users**: Add `-i` flag when piping stdin:

```bash
echo 'https://example.com' | docker run -v $PWD:/data -i archivebox/archivebox add
```

**Docker Compose users**: Add `-T` flag when piping stdin:

```bash
echo 'https://example.com' | docker compose run -T archivebox add
```

### Import from Files

ArchiveBox supports importing URLs from any text-based format:

```bash
# Generic text file with URLs
archivebox add < ~/Downloads/urls.txt

# RSS feed
archivebox add --parser=generic_rss < ~/Downloads/feed.xml

# With recursion depth
archivebox add --depth=1 'https://news.ycombinator.com#2020-12-12'
```

### Extract URLs from HTML/Markdown

ArchiveBox automatically extracts URLs from any text format:

```bash
echo 'any text with <a href="https://example.com">urls</a> in it' | archivebox add
```

## Browser Bookmarks & History

Export bookmarks or history from your browser in Netscape bookmark format:

<img src="https://github.com/ArchiveBox/ArchiveBox/assets/511499/32b494e6-4de1-4984-8d88-dc02f18e5c34" height="22px" style={{display: 'inline', verticalAlign: 'middle'}} /> **Browser Export Instructions**:

- **Chrome**: [Export bookmarks](https://support.google.com/chrome/answer/96816?hl=en)
- **Firefox**: [Export bookmarks](https://support.mozilla.org/en-US/kb/export-firefox-bookmarks-to-backup-or-transfer)
- **Safari**: [Export bookmarks](https://github.com/ArchiveBox/ArchiveBox/assets/511499/24ad068e-0fa6-41f4-a7ff-4c26fc91f71a)
- **Internet Explorer**: [Export favorites](https://support.microsoft.com/en-us/help/211089/how-to-import-and-export-the-internet-explorer-favorites-folder-to-a-32-bit-version-of-windows)
- **Opera**: [Export bookmarks](https://help.opera.com/en/latest/features/#bookmarks)

After exporting, import into ArchiveBox:

```bash
archivebox add < ~/Downloads/bookmarks.html
```

## MITM Proxy Integration

<img src="https://github.com/ArchiveBox/ArchiveBox/assets/511499/4f7bd318-265c-4235-ad25-38be89946b12" height="22px" style={{display: 'inline', verticalAlign: 'middle'}} /> **Real-time archiving via proxy**

Use [archivebox-proxy](https://github.com/ArchiveBox/archivebox-proxy) with [mitmproxy](https://mitmproxy.org/) to automatically archive all traffic from any device going through the proxy.

This provides [realtime archiving](https://github.com/ArchiveBox/ArchiveBox/issues/577) of all HTTP/HTTPS requests.

## Supported File Formats

ArchiveBox can parse URLs from:

- **Plain text** (`.txt`) - one URL per line
- **JSON** - various formats with URL fields
- **CSV** - columns containing URLs
- **RSS/Atom feeds** (`.xml`, `.rss`)
- **HTML bookmarks** (Netscape format)
- **Markdown** - extracts links from `[text](url)` format
- **SQL exports** - extracts URLs from database dumps
