For each web page you archive, ArchiveBox creates a **Snapshot** and preserves its content in multiple redundant formats. All files are saved as ordinary files on disk that you can access without needing ArchiveBox running.

## Output Files Explained

### Index Files

**`index.html`** - Human-readable page showing:
- Snapshot metadata (URL, timestamp, tags)
- Preview of archived content
- Links to all saved formats
- Status of each extraction method

**`index.json`** - Machine-readable metadata containing:
- All snapshot details
- Extraction results
- Timestamps and status codes
- File paths and sizes

### Web Content

**`singlefile.html`** - Created using [SingleFile](https://github.com/gildas-lormeau/SingleFile):
- Self-contained HTML file
- All CSS, images, fonts embedded as data URIs
- Works offline without external dependencies
- High fidelity preservation

**`output.html`** - DOM dump using headless Chrome:
- HTML after JavaScript execution
- Captures dynamically loaded content
- May include ads and tracking scripts

**`example.com/`** - wget clone:
- Mirrors the entire site structure
- Preserves links between pages
- Includes CSS, JS, images as separate files

**`warc/`** - Web ARChive format:
- Industry-standard archival format
- Used by Archive.org, libraries, institutions
- Contains raw HTTP requests/responses
- Long-term preservation format

### Visual Formats

**`screenshot.png`** - Visual snapshot:
- 1440x900 resolution
- Captures page as rendered in Chrome
- Useful for visual verification
- Shows layout and design

**`output.pdf`** - PDF version:
- Printable format
- Good for long-term readability
- Works on any device
- May lose some interactivity

### Text Extraction

**`article.html`** / **`article.json`** - Readable content:
- Uses Mozilla's Readability and Mercury
- Removes ads, navigation, footers
- Extracts main article text
- Preserves formatting
- Great for RSS readers or reading mode

### Metadata

**`title.txt`** - Page title extracted from `<title>` tag

**`favicon.ico`** - Site icon/favicon

**`headers.json`** - HTTP response headers:
- Content-Type
- Server information
- Cache headers
- Security headers

**`archive.org.txt`** - Archive.org permalink:
- ArchiveBox submits URLs to Archive.org by default
- Creates redundant backup
- Can be disabled with `SAVE_ARCHIVE_DOT_ORG=False`

### Media Files

**`media/`** - Downloaded media using [yt-dlp](https://github.com/yt-dlp/yt-dlp):
- Videos (MP4, WebM, etc.)
- Audio (MP3, M4A, etc.)
- Subtitles (SRT, VTT)
- Thumbnails
- Metadata JSON
- Supports 1000+ sites

### Source Code

**`git/`** - Git repository clone:
- Full commit history
- All branches
- Submodules
- LFS files (if configured)

## Configuration

You can enable/disable specific output formats:

```bash
# Disable specific extractors
archivebox config --set SAVE_PDF=False
archivebox config --set SAVE_SCREENSHOT=False
archivebox config --set SAVE_ARCHIVE_DOT_ORG=False

# View all extractor settings
archivebox config | grep SAVE_
```

Common settings:

- `SAVE_TITLE` - Extract page title
- `SAVE_FAVICON` - Download favicon
- `SAVE_WGET` - Create wget mirror
- `SAVE_WARC` - Create WARC file
- `SAVE_SINGLEFILE` - Create SingleFile HTML
- `SAVE_PDF` - Generate PDF
- `SAVE_SCREENSHOT` - Capture screenshot
- `SAVE_DOM` - Save DOM after JS execution
- `SAVE_READABILITY` - Extract article text
- `SAVE_MERCURY` - Extract article (alternative method)
- `SAVE_GIT` - Clone git repositories
- `SAVE_MEDIA` - Download media with yt-dlp
- `SAVE_ARCHIVE_DOT_ORG` - Submit to Archive.org

## Why Multiple Formats?

ArchiveBox saves content in multiple formats because:

1. **Redundancy** - If one format fails or becomes obsolete, others remain
2. **Fidelity** - Different formats capture different aspects (visual, interactive, text)
3. **Longevity** - Standard formats (PDF, WARC, HTML) will be readable for decades
4. **Accessibility** - Different formats suit different use cases (reading, sharing, analysis)
5. **No proprietary lock-in** - All formats are open standards or plain files

## File Access

All archived content is stored as regular files on disk:

```bash
# Browse directly
ls ./data/archive/
open ./data/archive/1234567890/index.html

# Or query the database
sqlite3 ./data/index.sqlite3 "SELECT url, timestamp FROM core_snapshot LIMIT 10;"

# Or use the Python API
archivebox shell
>>> from core.models import Snapshot
>>> Snapshot.objects.first().url
```
