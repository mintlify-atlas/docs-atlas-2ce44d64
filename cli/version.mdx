## Overview

The `version` command displays ArchiveBox version information, system metadata, and the status of all installed dependencies (binaries). This is essential for debugging and getting support.

## Basic Usage

```bash
archivebox version
```

```bash docker
docker compose run archivebox version
```

## What It Shows

### Version Information
- ArchiveBox version number
- Git commit hash (if installed from source)
- Build timestamp

### System Information
- Docker/QEMU status
- System architecture (x86_64, arm64, etc.)
- Operating system
- Platform details
- Python implementation and version

### Permissions & Filesystem
- User/group IDs (EUID, UID, PUID)
- Filesystem ownership (FS_UID, FS_GID)
- Filesystem permissions
- Atomic write enforcement status
- Remote filesystem detection

### Runtime Configuration
- Debug mode status
- TTY detection
- Sudo/root status
- Machine and collection IDs
- Search backend engine
- LDAP authentication status

### Binary Dependencies

When run inside a data directory, shows all detected binaries:
- Binary name
- Version
- Provider (pip, npm, brew, apt, env)
- Installation path
- Validation status (✓ valid or ✗ missing)

### Code & Data Locations

Shows paths to:
- ArchiveBox installation directory
- Configuration files
- Data directory
- Archive directory
- Database location

## Options

### `--quiet`, `-q`

Print only the version number, no other information.

```bash
archivebox version --quiet
# Output: 0.9.0
```

Equivalent to:
```bash
archivebox --version
```

### `--binaries`, `-b`

Show information for specific binaries only.

```bash
# Check Chrome and wget only
archivebox version --binaries=chrome,wget

# Check single binary
archivebox version --binaries=yt-dlp
```

## Use Cases

### Get Version for Bug Reports

When reporting issues, always include full version output:

```bash
archivebox version > version_info.txt
```

Include `version_info.txt` in your bug report.

### Check Installed Dependencies

Verify all required binaries are installed:

```bash
archivebox version
```

Look for ✗ marks indicating missing dependencies.

### Verify Chrome Installation

```bash
archivebox version --binaries=chrome
```

### Check Python Environment

Verify you're using the correct Python version and virtual environment:

```bash
archivebox version | grep PYTHON
# Output: PYTHON=Cpython (venv)
```

### Debugging Permission Issues

Check user/group IDs and filesystem permissions:

```bash
archivebox version | grep -E "(UID|PERMS)"
# Output: EUID=1000:1000 UID=1000:1000 PUID=1000:1000 FS_UID=1000:1000 FS_PERMS=644
```

### Scripting Version Checks

```bash
#!/bin/bash
VERSION=$(archivebox version --quiet)
if [[ "$VERSION" < "0.9.0" ]]; then
    echo "Please upgrade ArchiveBox"
    exit 1
fi
```

## Exit Codes

- **0** - Success, all dependencies valid
- **1** - One or more dependencies missing/invalid

## Troubleshooting

### No Binary Information Shown

**Cause:** Not running inside a data directory.

**Solution:** 
```bash
cd ~/archivebox/data
archivebox version
```

Or run `archivebox init` first:
```bash
archivebox init
archivebox version
```

### Missing Binaries

**Cause:** Dependencies not installed.

**Solution:**
```bash
archivebox install
```

### Permission Warnings

**Cause:** Mismatched user/group IDs.

**Solution:** Check the UID/GID output and ensure your data directory has correct permissions:
```bash
sudo chown -R 1000:1000 ~/archivebox/data
```
