## Release Channels

### Stable Release (`latest`)

The stable release is recommended for production use:

```bash
# Install stable via pip
pip install archivebox

# Install stable via Docker
docker pull archivebox/archivebox:latest
```

### Development Release (`dev`)

The development branch contains the latest features but may be unstable:

```bash
# Install dev via pip
pip install 'git+https://github.com/ArchiveBox/ArchiveBox@dev'

# Install dev via Docker
docker pull archivebox/archivebox:dev
```

### Specific Versions

You can install specific versions:

```bash
# Install specific version via pip
pip install archivebox==0.9.3

# Install specific version via Docker
docker pull archivebox/archivebox:v0.9.3
```

## Upgrading

### Upgrade Process

Always backup your data before upgrading:

```bash
# Backup your archive
cp -r ./data ./data.backup

# Upgrade ArchiveBox
pip install --upgrade archivebox
# or
docker pull archivebox/archivebox:latest

# Run migrations
cd data/
archivebox init
```

### Version Compatibility

ArchiveBox maintains backward compatibility:

- **v0.9.x** can read archives from v0.7.x, v0.8.x
- **v0.8.x** can read archives from v0.7.x
- **v0.7.x** can read archives from v0.4.x

Automatic migrations run when you execute `archivebox init`.

### Breaking Changes

Major breaking changes are documented in release notes. Always read the release notes before upgrading across major versions.

## Release Schedule

ArchiveBox follows semantic versioning:

- **Major versions** (x.0.0): Significant architectural changes
- **Minor versions** (0.x.0): New features, non-breaking changes
- **Patch versions** (0.0.x): Bug fixes, security updates

Release frequency:
- **Patch releases**: As needed for critical fixes
- **Minor releases**: Every few months
- **Major releases**: When significant changes warrant it

## Migration Guides

### Upgrading from v0.8.x to v0.9.x

1. Backup your data directory
2. Upgrade ArchiveBox: `pip install --upgrade archivebox`
3. Run migrations: `archivebox init`
4. Verify data integrity: `archivebox status`

### Upgrading from v0.7.x to v0.9.x

1. Backup your data directory
2. Upgrade to v0.8.x first (recommended)
3. Then upgrade to v0.9.x
4. Run migrations: `archivebox init`

### Upgrading from v0.4.x to v0.9.x

1. Backup your data directory
2. Upgrade to v0.7.x first
3. Then upgrade to v0.9.x
4. Run migrations at each step: `archivebox init`

For detailed migration instructions, see: [Upgrading or Merging Archives](https://github.com/ArchiveBox/ArchiveBox/wiki/Upgrading-or-Merging-Archives)
