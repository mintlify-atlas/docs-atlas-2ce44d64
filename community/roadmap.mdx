## Current Development Focus

### Plugin System Improvements

We're actively enhancing the plugin architecture to make ArchiveBox more extensible:

- Better plugin dependency management
- Improved plugin configuration
- Enhanced plugin testing and coverage
- More modular extractor plugins

### Performance & Scaling

Optimizing ArchiveBox for larger archives:

- Database query optimization
- Better handling of large crawls
- Improved concurrency and parallelization
- More efficient storage management

### User Experience

Making ArchiveBox easier to use:

- Enhanced web UI with better navigation
- Improved CLI output and feedback
- Better error messages and debugging
- Simplified configuration

## Planned Features

### JavaScript Execution During Archiving

**Status**: Planned

Run custom JavaScript during archiving to:

- Block ads and tracking scripts
- Dismiss cookie popups and modals
- Auto-scroll infinite feeds
- Expand comment threads
- Close overlays and interstitials

Follow progress: [Issue #51](https://github.com/ArchiveBox/ArchiveBox/issues/51)

### Multi-Snapshot Improvements

**Status**: Planned

Better support for saving multiple snapshots of the same URL:

- Native multi-snapshot tracking without hash-date workaround
- Visual diff comparison between snapshots
- Change detection and alerting
- Timeline view of changes

Follow progress: [Issue #179](https://github.com/ArchiveBox/ArchiveBox/issues/179)

### Enhanced Crawling

**Status**: In Development

More advanced crawling capabilities:

- Better depth control and scoping
- Intelligent crawl scheduling
- Domain and path filtering
- Crawl budget management
- Integration with advanced crawlers (Browsertrix, Scrapy)

### Federation & Sharing

**Status**: Planned

Share portions of your archive with others:

- Federated archive discovery
- Selective sharing with other ArchiveBox instances
- Contribution to central archives (Archive.org integration)
- Collaborative archiving projects

### REST API Enhancements

**Status**: Alpha ‚Üí Beta

Improving the REST API:

- Complete API coverage for all features
- Better documentation and examples
- API versioning and stability
- Webhooks for events
- GraphQL support consideration

Current API: https://demo.archivebox.io/api (alpha)

## Extractors & Integrations

### New Extractors in Development

- **Better audio/video handling** - Enhanced media extraction
- **Social media improvements** - Better support for Twitter, Reddit, etc.
- **Document formats** - Enhanced PDF, DOCX, etc. archiving
- **Email archiving** - Archive email threads and attachments

### Platform Integrations

- **Cloud storage** - Better S3, B2, Google Drive integration
- **Search backends** - Enhanced FTS5, Sonic, Elasticsearch support
- **Authentication** - More SSO/OAuth providers
- **Monitoring** - Prometheus metrics, logging improvements

## Technical Improvements

### Database & Storage

- **PostgreSQL support** - Optional PostgreSQL backend (in addition to SQLite)
- **Object storage** - Better remote storage handling
- **Deduplication** - Content deduplication across snapshots
- **Compression** - Automatic compression of archived content

### Security & Privacy

- **End-to-end encryption** - Encrypted archive storage
- **Access controls** - Fine-grained permissions
- **Audit logging** - Comprehensive activity logs
- **Security hardening** - Enhanced security defaults

### Developer Experience

- **Better Python API** - More Pythonic interfaces
- **Type hints** - Complete type coverage
- **Documentation** - Comprehensive API docs
- **Testing** - Higher test coverage
- **CI/CD** - Faster, more reliable builds

## Release Timeline

We don't commit to specific release dates, but typical cadence:
